# 竞技场榜单化与成绩导出增强 Requirements

## Overview

当前“竞技场”里的热力图不够实用，用户更需要的是一个能从多个维度直接比较队员成绩的排行榜系统。同时，当前成绩导出仍然停留在原始数据层，无法快速体现队员的突出成绩、国家达标等级、目标里程碑和短期进步表现。

这次需求拆成两个互相关联但可以独立落地的方向：

1. 把竞技场从热力图改成多维排行榜。
2. 把成绩导出升级成更像“成绩简报模板”的导出结果。

## Problem Frame

### A. 竞技场热力图的问题

热力图有“看起来像市场”的感觉，但对家长、教练和学员本人来说，真正想知道的是：
- 在某个项目里，谁排名最高
- 同项目、同性别、同池长下，前几名分别是谁
- 如果按年龄段看，这个孩子在同龄人里排第几
- 如果不分年龄，这个孩子在全体里排第几

热力图对这些问题的回答不够直接。用户最终还是要把注意力落回“榜单”和“差距”。

### B. 当前导出的问题

现在导出的成绩更像原始流水，适合系统归档，不适合给家长、教练或外部人员快速理解一个孩子的突出表现。它缺少：
- 达标国家等级
- 达成过的目标/里程碑
- 最近一段时间进步了多少
- 哪些项目特别强
- 哪些成绩最值得被重点看到

## Users

- 家长：想一眼看懂孩子最强项目、最近进步、当前等级。
- 教练：想快速看某个项目、某个年龄段、某个性别里的排位。
- 管理员：想导出更像“成绩名片”或“成绩简报”的内容，而不是只导出流水。

## Goals

- 竞技场改成多维排行榜系统，替代当前热力图主视图。
- 排行必须严格发生在“同项目、同池长、同性别”的边界内。
- 排行可以支持按年龄段或不区分年龄切换。
- 导出结果能够突出队员的代表性成绩与成长亮点，而不是只给明细数据。
- 所有新页面和组件继续完全融入当前主色调、UIUX 和 shadcn 组件体系。

## Non-Goals

- 不做跨项目统一综合评分。
- 不做实时榜单刷新。
- 不把导出模板做成复杂排版编辑器。
- 不把竞技场重新做回图表优先页面。

## Key Product Decisions

### 1. 竞技场的主视图改成“排行榜市场”

竞技场仍然保留“市场”心智，但不再靠热力图表达，而是靠多维榜单矩阵表达。

核心结构建议：
- 顶部筛选：
  - 性别：男子 / 女子 / 混合
  - 池长：短池 / 长池 / 全部
  - 年龄：按年龄段 / 不区分年龄
  - 队伍：全部 / 某队
- 中部主榜：
  - 先按项目列出赛道卡片
  - 每个赛道卡片里显示 Top N 队员
  - 直接展示排名、最佳成绩、领先差距
- 侧边或下方详情：
  - 当前选中赛道的完整榜单
  - 排名变化、领先优势、近期进步

### 2. “男女混”不是跨性别直接比，而是一个显式模式

当用户选择“男女混”时，要明确说明这是“公开总榜视角”，不是标准比赛分组。默认仍然建议优先展示男子 / 女子分开榜单。

### 3. 年龄维度建议做成两级模式

- 不区分年龄：看全体公开选手总榜
- 年龄段：按出生年份映射到年龄桶，例如：
  - U8
  - U10
  - U12
  - U14
  - U16+

这样用户更容易理解，而不是直接裸露出生年份筛选。

### 4. 导出要从“流水导出”升级成“成绩简报导出”

导出建议新增一类“成绩简报模板”，至少包含：
- 基本信息：
  - 队员名
  - 队伍
  - 年龄 / 年龄段
- 成绩亮点：
  - 当前最强 3 个项目
  - 当前 PB
  - 最近 X 天最大进步项目
- 等级与里程碑：
  - 当前国家等级 / 下一达标线
  - 已达成目标
  - 距离下一目标还差多少
- 训练趋势摘要：
  - 最近 30 天进步秒数
  - 最近 90 天进步秒数

### 5. 导出形式不建议只停留在 CSV

如果只是把更多字段塞进 CSV，依然不够“让人家更容易了解队员的突出成绩”。

推荐做两层：
- 保留原始 CSV 导出，给系统/教练处理数据
- 新增“成绩简报模板”导出，用于阅读、转发、截图、打印

## Recommended Direction

### 方向一：竞技场榜单化

推荐把竞技场做成“赛道榜单看板”而不是热力图。  
这是对当前竞技场最实用的升级，也最符合用户真实使用目标。

### 方向二：导出模板报告化

推荐新增“成绩简报模板”导出，而不是只增强 CSV 列。  
因为用户需要的是“更容易理解”，不是“更多原始列”。

## Alternative Considered

### A. 保留热力图，只在旁边补榜单

不推荐作为主方向。  
因为用户已经明确觉得热力图“没啥用”，继续保留它为主视图只会让页面继续分散注意力。

### B. 只增强 CSV，不做简报模板

不推荐。  
增强 CSV 只能让系统更完整，不能让外部阅读体验显著提升。

## Success Criteria

- 用户打开竞技场后，可以直接按赛道看到清晰的排名结果，而不是先理解热力图。
- 排名切换性别、池长、年龄段时，比较边界始终正确。
- 导出结果能在一页内让人快速理解该队员最突出的成绩与进步亮点。
- 新页面、新卡片、新导出视图在风格上与现有公开页和后台页完全融为一体。
