# 后台队员移动端恢复与日期弹层统一 Requirements

## Overview

当前后台 `队员管理` 页在桌面端可用，但移动端已经退化成不可正常浏览成员列表；同时 `队员管理 / 成绩管理 / 目标管理` 这些使用日期组件的页面，其弹出的选择面板样式与当前 shadcn/Radix 主题不一致，显得突兀。这轮需求是把移动端可用性补回去，并把日期组件统一到更标准的 `Calendar + Popover` 组合风格。

## Problem Frame

### A. 队员管理页移动端退化

后台 `队员管理` 页之前具备移动端可用性，但当前版本下，页面虽然显示“当前显示 2 名成员”这类计数，说明接口已经返回了数据，实际移动端列表却没有正确渲染出来。  
这意味着问题不在接口，而在页面的移动端展示层本身。

### B. 日期组件弹层风格不统一

当前后台几个页面虽然已经统一改成日期选择器输入，但弹出的日历/年份面板在视觉上仍然不像当前系统里其它 shadcn/Radix 组件：
- 弹层外观与 Select / Card / Popover 的表面层级不完全一致
- 结构虽然可用，但细节像“另外拼出来的组件”

用户要的不是换功能，而是把这类弹层继续收口成一套一致的组件语言。

## Users

- 管理员：希望在手机端也能正常管理队员，不需要缩放或横向拖动才能看列表。
- 维护者：希望日期选择器的弹层风格和当前 shadcn/Radix 主题统一，不再出现“控件能用但样式怪”的情况。

## Goals

- 恢复后台 `队员管理` 页的移动端成员列表可用性。
- 保持桌面端现有交互与视觉风格不变。
- 统一后台日期组件弹层到当前 shadcn/Radix 风格。
- 保证 `队员管理 / 成绩管理 / 目标管理` 使用同一套日期弹层表面语言。

## Non-Goals

- 不重做后台整体布局。
- 不更换日期选择功能的业务逻辑。
- 不把队员管理桌面端表格改成全新的结构。

## Key Product Decisions

### 1. 队员管理采用“桌面表格 + 移动端卡片”双视图

推荐恢复方式：
- `xl` 以上保留当前桌面端表格布局
- `xl` 以下恢复卡片式成员列表
- 两种视图展示相同数据，只是信息密度不同

### 2. 日期组件统一以 `Calendar + Popover` 为主

当前日期选择器应继续建立在：
- 共享 `Calendar` primitive
- 共享 `Popover` primitive

而不是保留一个“功能上可选日期，但弹层样式像独立小部件”的状态。

### 3. 不改变其它页面的整体交互风格

这次修复的原则是：
- 修复移动端可用性
- 修复弹层样式突兀
- 不主动改变桌面端布局、按钮行为和主要交互路径

## Success Criteria

- `队员管理` 页在移动端可以稳定看到已录入成员列表。
- 移动端成员列表不会超出一屏宽度。
- 日期组件弹层的外观和当前 `Select / Card / Popover` 风格一致。
- `队员管理 / 成绩管理 / 目标管理` 的日期组件弹层保持统一。
