# 竞技场详情化与公开分享修正 Requirements

## Overview

当前竞技场页的“赛道排行榜”和“赛道详情”存在明显职责重叠，公开个人详情页的模块纵向节奏也不够协调，且“保存分享海报”只截取了顶部首屏内容，无法形成真正可分享的完整成长卡片。这个需求要把竞技场进一步收束成“详情优先”的单一主视图，同时把公开个人详情页的布局节奏与分享能力补齐。

## Problem Frame

### A. 竞技场的信息层次重复

现在竞技场左侧“赛道排行榜”已经展示了分组标签、Top 3 和时间差，右侧“赛道详情”又展示了更完整的同一组信息。两块并排之后，用户需要先理解两个相似模块分别做什么，结果却没有获得成倍增加的信息价值。

用户真正需要的是：
- 在同项目、同池长、同性别的边界里，快速看懂当前这条赛道的完整竞争态势。
- 一个主视图内同时看到头名、差距、热度、Top 排行、分组标签和关键信号。
- 不需要先扫左侧缩略版，再去右侧看完整版。

### B. 竞技场顶层指标中“焦点赛道”价值弱

现在“焦点赛道”只是取当前筛选结果中的第一个分组名称，既不跟随用户当前选中的赛道，也没有补充竞争信息，体感上只是重复显示一遍项目名，占用了宝贵的指标位置。

### C. 公开个人详情页的视觉节奏和分享体验不完整

公开个人详情页顶部主卡、三枚成绩指标卡、成长分析模块之间的上下间距不统一，视觉上显得挤压。更严重的是，“保存分享海报”只截取了顶部 `shareRef` 容器，因此导出的图片没有成长曲线、达标差距、目标里程碑等真正有分享价值的内容。

## Users

- 家长：希望竞技场一眼看懂当前赛道格局，也希望公开成长页导出的分享图足够完整。
- 教练：希望减少重复信息，把页面注意力集中在真正有判断价值的赛道详情上。
- 学员：希望公开分享图能完整体现自己的成长，而不是只有头像区和几个标题。

## Goals

- 把竞技场收束成一个信息更完整的详情主视图，减少重复模块。
- 删除当前价值较弱的“焦点赛道”指标卡，用更有用的指标替代。
- 让公开个人详情页的模块垂直间距统一，形成稳定的页面节奏。
- 让“保存分享海报”导出整个公开成长页的核心内容，而不是只截取第一块头图。
- 所有改动继续保持当前主色调、shadcn/ui 风格和移动端兼容性。

## Non-Goals

- 不重新做竞技场的产品方向，不回退到热力图。
- 不把个人详情页改造成新的独立分享路由产品，本轮优先修正现有详情页导出。
- 不新增社交分享 SDK、外链发布或复杂模板编辑器。

## Key Product Decisions

### 1. 竞技场改为“详情优先”的单一主体验

推荐方向：
- 保留赛道分组浏览能力，但不再让“缩略排行榜”和“详情排行”并排重复出现。
- 页面主区域直接以“赛道详情卡”作为核心内容。
- 分组切换可以改成更轻量的列表、切换条或概览卡，不再承担重复展示 Top 3 的责任。

### 2. 竞技场顶层指标替换“焦点赛道”

“焦点赛道”应被删除，替换成对当前筛选更有解释力的指标。优先级更高的候选指标是：
- 当前选中赛道的热度等级
- 当前选中赛道的头名优势
- 当前筛选下的平均竞争人数
- 当前筛选下的最高热度分组标签

推荐优先显示“当前选中赛道热度”或“头名优势”，因为这两个指标最能帮助用户理解赛道格局。

### 3. 公开个人详情页建立统一纵向节奏

页面中的主模块之间应采用统一的垂直间距体系。例如：
- 顶部主卡 与 指标卡：统一间距
- 指标卡 与 成长分析视图：统一间距
- 成长分析内部各大模块继续使用同一间距尺度

重点不是变大或变小，而是上下节奏一致。

### 4. 分享海报导出必须覆盖完整成长内容

“保存分享海报”不应只截取顶部 Hero 卡。导出的内容至少要覆盖：
- 顶部队员信息与项目选择结果
- PB / 记录数 / 目标进度三枚指标卡
- 成长曲线
- 达标差距摘要
- 等级/目标模块

如果整页高度过长，可以改成“分享版容器”，但必须是完整内容的专用组合，而不是只截首屏。

## Recommended Direction

推荐做成 **“竞技场详情主视图 + 公开成长页完整分享容器”**。

原因：
- 它直接消除了竞技场的重复信息问题。
- 它给顶层指标腾出了更高价值的位置。
- 它修正的是用户当前已经明显感知到的问题，而不是继续增加功能点。
- 它与现有公开个人详情页的成长表达是同一条产品逻辑，形成更完整的外部展示体验。

## Alternative Considered

### A. 保留赛道排行榜和赛道详情，只微调文案

不推荐。

因为信息结构重复本身没有解决，用户仍要在两个相似模块之间切换注意力。

### B. 删除赛道详情，只保留赛道排行榜

不推荐。

因为当前更有价值的是赛道详情，排行榜只是它的缩略版，不应该保留缩略版而删除完整版。

### C. 个人详情页继续只截首屏，再额外引导用户去独立分享页

不推荐作为本轮主解。

因为用户当前点击的是“保存分享海报”，预期就是在当前页得到一张完整图，而不是再跳去第二个页面才有完整内容。

## Success Criteria

- 竞技场页面不再同时出现两个承担相似职责的大模块。
- 当前选中赛道的核心竞争信息在一个主模块里即可读完。
- 顶层指标中不再出现解释力弱的“焦点赛道”。
- 公开个人详情页的主模块上下间距统一，桌面和移动端都显得协调。
- 导出的分享海报包含完整成长关键信息，而不是只有顶部头图。
