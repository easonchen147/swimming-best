# 后台花名册与时间选择器收口 Requirements

## Overview

这次需求聚焦后台几个高频页面的最后一段“使用感毛边”：

- 队员管理右侧表格在桌面端列宽分配不合理，导致出现横向滚动条；
- 队员管理里“公开状态”列的模式文案仍然是英文枚举值，不符合中文产品语境；
- 队伍管理卡片还在显示 `ORDER: 1` 和 `ACTIVE`，信息价值低且语言不自然；
- 队员管理、成绩管理、目标管理等页面共用的日期选择弹层层次过厚，看起来像多个组件叠在一起。

这轮不是新增能力，而是把后台主工作流进一步收口到更稳定、更干净的状态。

## Problem Frame

### A. 队员表格列宽过度偏向姓名列

当前队员管理表格第一列留得过宽，右侧列在桌面端被压缩，最终把整个表格推到了需要横向滚动的状态。对于一个后台花名册页，这种滚动是纯损耗。

### B. 公开状态模式文案直接暴露英文枚举

`nickname`、`real_name`、`hidden` 这些值本来是内部状态，不应该原样出现在用户界面里。当前显示既不自然，也增加了理解成本。

### C. 队伍卡片展示了低价值元数据

“已管理队伍”卡片里展示 `ORDER: 1` 不够直观。管理员更关心的是“这个队现在有多少队员”，因为这能直接反映使用情况。`ACTIVE` 也应该改成中文环境下更自然的“有效”。

### D. 共享日期弹层视觉层级过厚

当前日期选择器用了自定义外层容器、标题区、按钮区，再叠加 Calendar 内部结构，导致弹层像两个组件叠在一起。你给出的官方 date picker demo 不会这样，这说明当前实现偏离了推荐结构。

## Users

- 教练 / 管理员：频繁浏览队员、队伍、成绩、目标页面，需要更直观的信息布局和更干净的日期交互。
- 维护者：希望共享日期组件的结构更接近官方推荐，避免后续再出现多页一起发散。

## Goals

- 去掉队员管理表格在桌面端的不必要横向滚动。
- 把公开状态列里的模式文案改成中文展示。
- 队伍管理卡片用“队员数”替代 `ORDER`，并把状态文案改成“有效”。
- 修复共享日期弹层的层级和视觉结构，让后台多个页面的时间组件都变得干净一致。

## Non-Goals

- 不重做整个后台页面布局。
- 不调整队伍排序逻辑本身，`sortOrder` 仍然保留在数据层和表单里。
- 不新增新的时间相关产品能力，只修正现有组件的呈现和信息表达。

## Key Product Decisions

### 1. 队员表格优先保证桌面端无横向滚动

推荐方向：

- 缩窄姓名列；
- 给状态和操作列更明确的紧凑宽度；
- 让表格优先适配常见桌面宽度，而不是为了首列内容过度留白。

### 2. 界面只显示用户能直接理解的中文状态

推荐方向：

- `hidden` 不再额外显示模式标签；
- `nickname` 显示为“昵称”；
- `real_name` 显示为“真名”。

### 3. 队伍卡片用“队员数”替代排序号展示

排序号仍然是后台维护字段，但不必出现在主要浏览卡片里。  
主列表更应该优先展示“有多少队员”，这比 `ORDER` 更有运营价值。

### 4. 日期弹层收回到官方 date picker 结构

推荐方向：

- 共享 `DatePickerInput` 尽量贴近官方 `Popover + trigger + Calendar` 组合；
- 弹层外壳改为更轻的单层结构；
- 避免标题条、额外内层卡片、重边框和重复背景造成的“重叠感”。

## Recommended Direction

推荐把这轮工作定义为 **“后台高频浏览与选择器精修”**：

- 队员管理：修表格密度和中文标签；
- 队伍管理：修卡片信息优先级；
- 共享日期组件：统一收回到官方推荐结构。

这是低风险、高体感收益的一轮修正，而且都集中在后台管理员日常最常碰到的页面。

## Success Criteria

- 队员管理表格在常见桌面宽度下不再出现丑陋的横向滚动条。
- 公开状态模式文案只显示中文“昵称 / 真名”，隐藏状态不额外显示英文标签。
- 队伍管理卡片显示队员数，状态文案使用“有效”。
- 队员管理、成绩管理、目标管理等页面的日期弹层不再出现明显的双层叠加感，整体接近官方 demo 的单层清爽结构。
