# 页面风格与筛选链路完整性 Requirements

## Overview

当前系统的主要问题已经从“有没有功能”转到“页面是否统一、交互是否完整、筛选是否真生效”。这次需求聚焦 4 个方面：公开个人成绩详情页的上下节奏统一、把“保存分享海报”升级为整页成长长图导出、校验并统一前端页面的视觉与 Radix/shadcn 组件风格、以及把所有可见搜索/筛选控件补成真正的前后端一致能力。

## Problem Frame

### A. 公开个人成绩详情页的上下节奏不一致

当前公开个人成绩详情页已经有 Hero、成绩指标卡和成长分析区，但它们之间的垂直间距并不完全统一。用户感知上会变成“前面几行挤得紧，后面几行又松”，破坏页面的秩序感，也会让整页截图的观感不够统一。

### B. “保存分享海报”还不是真正的整页成长图

当前分享导出虽然已经扩大到比首屏更完整的内容，但产品目标应该更明确：用户点击保存时，得到的是“这个公开成长页”的长图版本，而不是某个局部卡片的截图。也就是说，导出行为应以整页成长档案为语义，而不是以局部模块为语义。

### C. 各页面的组件风格和主题皮肤仍有漂移

当前项目已经以 shadcn/Radix 作为共享组件基础，但仍有部分页面或表单控件使用原生控件、缺少 Group 语义包裹，或者在 spacing / tone 上和系统其他页面不完全一致。这会让用户在不同页面间切换时感到风格断裂。

### D. 搜索和筛选链路没有全部打通到接口层

目前多个页面上的搜索框和筛选控件确实能产生“看起来像搜索”的效果，但不少还只是前端本地过滤，接口层并没有统一支持这些入参。这样一来，一旦数据量变大，或者未来换页、分页、分享链接透传条件，就很容易出现“前端有输入框，后端却没真正识别筛选条件”的问题。

## Users

- 家长：希望公开成长页既好看又能一键保存成完整长图。
- 教练/管理员：希望管理后台的搜索、筛选是真正可靠的，不是摆设。
- 维护者：希望前端页面风格、组件语义和接口参数保持长期一致，避免以后继续出现样式和筛选漂移。

## Goals

- 统一公开个人成绩详情页主模块之间的纵向节奏。
- 把“保存分享海报”升级为整页成长长图导出。
- 把可见页面里的关键控件进一步统一到 Radix/shadcn 组件语义和当前主题皮肤上。
- 让主要列表页与公开首页的搜索/筛选能力前后端一致，接口真正识别这些参数。
- 确保所有可见搜索框都具有真实搜索能力，而不是纯装饰输入框。

## Non-Goals

- 不重做产品视觉主题，不改主色调方向。
- 不新增复杂的分享模板编辑器或社交平台 SDK。
- 不做全文搜索引擎或高级查询语言。
- 不把所有前端页面都改成服务端搜索优先；只处理当前可见且明确存在搜索/筛选交互的主要页面。

## Key Product Decisions

### 1. 公开成长页按“整页档案”导出，而不是按“单模块截图”导出

推荐方向：
- 导出语义升级为“整页成长长图”。
- 截图容器应覆盖 Hero、项目切换、成绩指标卡和成长分析区。
- 顶部导航与明显的交互按钮可以继续排除在导出之外，但导出主体必须是整页成长内容。

### 2. 页面上下节奏由页面骨架统一控制

不再让各个子模块自己决定外层间距，而是让页面骨架决定主区块之间的统一 gap。  
这样 Hero、指标卡、成长分析模块在桌面和移动端都能保持相同节奏。

### 3. 样式统一的重点是“共享语义层”，不是纯表面改色

真正需要统一的是：
- Checkbox / Select / Label 等共享表单 primitive 的使用方式
- SelectItem 是否包在 SelectGroup 中
- 是否还残留原生 checkbox 破坏整体皮肤
- 页面 spacing、背景层次、交互态是否仍在同一套设计语言中

这比简单地调整几个颜色值更重要。

### 4. 搜索和筛选要前后端一起闭环

推荐把以下页面的列表能力补成接口可识别的查询参数：
- 公开首页队员列表
- 管理后台队伍列表
- 管理后台队员列表
- 管理后台项目列表

前端页面仍可保留即时输入体验，但实际展示结果应来自真正识别 `search` / `teamId` 等参数的接口响应，而不是只靠本地数组过滤。

## Recommended Direction

推荐把这轮工作视为 **“公开成长页导出升级 + 列表查询完整化 + 共享 UI 语义统一化”** 的一次集中修正。

原因：
- 这几件事本质上都在解决“看起来有，但不够完整”的一致性问题。
- 它们互相补强：页面节奏统一后，整页导出才真正好看；控件语义统一后，筛选/搜索行为也更容易长期稳定。
- 这类修正越早做，后续新增页面越不容易继续漂。

## Alternative Considered

### A. 只改公开成长页导出，不碰样式和筛选链路

不推荐。

因为用户已经明确指出了“整页节奏”和“筛选参数摆设”的问题，只修导出会留下结构性不一致。

### B. 只做前端本地过滤增强，不补后端参数

不推荐。

因为这只能改善当前小数据量体验，不能解决参数语义不一致的问题。

### C. 彻底做一次全站重构

不推荐。

因为这轮目标是高价值收口，不是重开设计系统工程。

## Success Criteria

- 公开个人成绩详情页的主模块上下间距读起来是统一的。
- “保存分享海报”导出的图片可以代表整页成长档案，而不是局部卡片。
- 主要页面里的 Checkbox / Select 等控件继续向统一的 Radix/shadcn 语义收敛。
- 公开首页、管理后台队伍/队员/项目页面的搜索筛选都能通过接口层真正识别并返回过滤结果。
- 现有主色调、主题皮肤和整体 UIUX 风格保持一致，没有出现新的视觉分叉。
