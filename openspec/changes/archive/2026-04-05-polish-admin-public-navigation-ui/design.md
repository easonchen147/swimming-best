## Context

当前前端已经有一套共享的 `Button`、`Card`、`Select` 与布局壳层，但高频动作区仍然存在“组件统一、表达不统一”的问题。管理后台头部同时存在按钮和伪按钮，退出登录按钮放在侧栏底部且没有接真实交互；侧边栏只有部分导航项带图标，造成信息密度和扫描节奏不一致；后台首页“快速操作”使用深色渐变卡片，与旁边的流程面板不属于同一视觉语义。公开页顶部导航则仍然更像文案链接，对比页还会在用户没有明确操作前自动选中队员并开始加载结果。

这轮实现的约束很明确：不改当前产品主色体系，不新增新的组件库，不改后端接口，只通过现有共享组件和布局逻辑，把动作入口、空状态和移动端排布统一到更清晰的一层。

## Goals / Non-Goals

**Goals:**
- 让管理后台头部动作、侧边栏导航、首页快捷入口使用一致的按钮与图标语言。
- 让公开页顶部导航变成真正的按钮组，而不是样式偏弱的链接。
- 让对比页进入“显式选择”模式，只有在满足条件时才请求并显示对比结果。
- 保证桌面和移动端都能稳定看到主要入口、理解当前状态并执行下一步动作。

**Non-Goals:**
- 不调整品牌主色、排版基调或整站视觉方向。
- 不新增后端接口，也不改变 SQLite / Flask 数据层。
- 不重做对比页的数据模型，只改前端默认选择和展示流程。
- 不扩展新的导航模块或新的后台页面。

## Decisions

### 1. 头部动作统一复用共享按钮语义
**Decision**: 管理后台头部将“查看公开页”与“退出登录”都改为共享 `Button` 组件；公开页头部的“首页 / 对比 / 管理后台”也统一使用按钮形式，但保留不同的色调区分。
**Rationale**: 用户最常接触的是头部动作，必须先让这些入口的交互 affordance 一致。继续使用原生 `button` 或 ghost 文案链接会让视觉优先级和可点性判断不稳定。
**Alternative considered**: 为不同区域分别写定制按钮样式。被拒绝，因为会继续扩大按钮实现分叉。

### 2. 侧边栏采用“全量图标 + 高亮白字”策略
**Decision**: 为后台每个导航项补齐 Lucide 图标，并维持选中项“高亮底色 + 白色图标/文字”的表达。
**Rationale**: 后台信息架构已经固定，图标缺失会打断扫描节奏。补齐图标后，桌面和移动菜单都能共享同一套导航心智。
**Alternative considered**: 只给高频入口补图标。被拒绝，因为不完整的图标体系依旧会制造不一致。

### 3. 快速操作面板回到轻表面卡片
**Decision**: 后台首页“快速操作”卡片改为与页面主体一致的浅色表面，三个快捷入口使用高可读的 outline/button 组合，而不是深色渐变卡片中的浅字按钮。
**Rationale**: 这里承载的是操作，不是品牌展示。与“推进管理流程”保持同一层级的表面语义，能提升可读性并减少视觉割裂。
**Alternative considered**: 保留深色卡片，仅调整文字颜色。被拒绝，因为问题不只是颜色，而是整块面板语义与周围布局脱节。

### 4. 对比页改为显式选择驱动
**Decision**: 页面首次加载只获取公开队员列表，不默认选中任何队员；事件列表基于“第一个已选队员”加载，但不自动选择项目；只有当用户至少选中两位队员且明确选择项目后，才发起 compare 请求。
**Rationale**: 当前默认选中和自动加载会让页面像“系统替用户做了决定”，不符合对比场景的操作预期。显式选择能让空状态和结果态边界更清楚。
**Alternative considered**: 保留默认选中，但隐藏加载文案。被拒绝，因为根因不是文案，而是默认决策本身。

### 5. 空状态与加载状态分离
**Decision**: 对比页引入独立的空状态文案与结果加载状态。未选满两个队员时显示“待选择对比的队员”，已选队员但未选项目时显示项目提示，只有真正请求中才显示加载反馈。
**Rationale**: 用户需要知道“现在缺什么条件”，而不是一上来就看到“加载中”。
**Alternative considered**: 保持统一 `LoadingState`。被拒绝，因为 loading 与 missing-selection 是两种不同状态。

## Risks / Trade-offs

- [Risk] 对比页事件列表依赖第一个已选队员，若切换第一个队员会重置项目选择 → Mitigation: 在交互上明确重置 `selectedEventId` 与结果区，让状态转换可预期。
- [Risk] 头部按钮增多后移动端空间变紧 → Mitigation: 使用响应式换行/网格布局，保证按钮保持足够点击面积。
- [Risk] 补齐侧边栏图标后视觉密度上升 → Mitigation: 复用现有小尺寸 Lucide 图标，并保持与现有文字间距和 hover/active 动画一致。
