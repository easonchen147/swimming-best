## Context

当前竞技场已经有按赛道聚合的公开成绩数据和右侧赛道详情，但主视图仍以热力图为核心。用户明确反馈热力图“没啥用”，真正需要的是更直接的排名统计能力，用来从多个维度看同项目内谁更强、差距多大、同龄人里排第几。

这意味着竞技场不应该继续围绕图形可视化修修补补，而应该转成榜单产品。赛道聚合后端可以复用，但前端主视图和部分数据契约需要改成排行榜优先。

## Goals / Non-Goals

**Goals:**
- 去掉热力图主视图，改成榜单化竞技场。
- 支持性别、池长、年龄段、队伍多维筛选。
- 保持比较边界在同项目、同池长内。
- 继续沿用当前 UI 主题和组件体系。

**Non-Goals:**
- 不做跨项目综合评分。
- 不做实时榜单。
- 不引入新的图表库。

## Decisions

### 1. 主视图改为分组榜单卡片
**Decision**: 竞技场主区域渲染多个“赛道榜单卡片”，每个卡片展示一个项目赛道的 Top N。
**Rationale**: 这比热力图更直接，也更符合你想要的“多方位对比队员成绩”。

### 2. 年龄维度以后端年龄桶返回
**Decision**: 年龄段逻辑放在后端计算，前端只消费 `ageBucket` 与 `ageMode`。
**Rationale**: 这样前后端口径统一，不会出现 UI 上分桶和后端聚合不一致的问题。

### 3. 男女混榜作为显式模式
**Decision**: `gender=all` 视为“公开总榜视角”，不再等同于比赛标准分组。
**Rationale**: 满足用户要“男女混”的需求，同时不误导默认竞技分组。

## Risks / Trade-offs

- [Risk] 榜单卡片太多导致页面过长 -> Mitigation: 默认只显示 Top 赛道卡片，详情区承接更多信息。
- [Risk] 年龄桶切换让用户困惑 -> Mitigation: 文案显式标记“按年龄段 / 不分年龄”。
- [Risk] 现有热力图相关代码残留造成维护负担 -> Mitigation: 本轮清理主视图引用，不让热力图继续作为主路径存在。
