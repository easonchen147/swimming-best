## Context

当前公开个人详情页已经有完整成长内容，但导出行为和区块节奏还不够稳定；同时，多个主要列表页的搜索与筛选仍然主要停留在前端本地过滤层。项目也已经确立了 shadcn/Radix 作为共享组件语义层，但后台高频表单中仍残留原生 checkbox 与不完整的 Select 组合。

这次设计要同时解决 UI 层和接口层的一致性问题：一边把 visible controls 收口到统一 primitive 语义，一边让这些输入在后端列表查询中真正生效。

## Goals / Non-Goals

**Goals:**
- 让公开成长页成为真正可导出的整页长图。
- 让主要列表页搜索/筛选具备真实的后端查询支持。
- 让关键表单控件进一步统一到共享 Radix/shadcn 语义。
- 保持当前主色调与页面风格稳定。

**Non-Goals:**
- 不引入新的搜索服务或分页系统。
- 不调整成绩分析、达级或目标计算逻辑。
- 不做全站彻底重构。

## Decisions

- **列表查询统一增加 `search` 参数**
  - 这是解决“输入框有了但接口没有识别”的最直接方式。
  - 不选择只靠前端本地过滤，因为这无法保证长期的前后端一致性。

- **公开首页采用“全量选项源 + 条件结果集”双状态**
  - 公开首页仍然需要稳定的队伍筛选按钮，因此保留一份全量队伍来源。
  - 当前可见队员列表则来自搜索/队伍条件的真实接口响应。

- **新增共享 Checkbox primitive**
  - 使用共享 Radix checkbox 而不是继续放任页面内的原生 checkbox，有助于主题统一和长期维护。

- **长图导出显式传入尺寸**
  - 通过 `scrollWidth`/`scrollHeight` 与 `canvasWidth`/`canvasHeight` 显式控制输出尺寸，减少长图裁切和缩放失真风险。

## Risks / Trade-offs

- **列表页请求频率上升** → 使用 deferred 搜索值，避免每次键入都立即打满请求。
- **公开首页按钮来源与当前结果集分离后状态复杂度上升** → 将“队伍选项源”和“当前列表结果”拆成清晰的两个状态。
- **Checkbox 组件替换可能影响原表单行为** → 通过页面级测试覆盖 boolean 提交流程。
- **长图尺寸变大可能增加导出时间** → 控制像素比，并只在必要区域内导出。
