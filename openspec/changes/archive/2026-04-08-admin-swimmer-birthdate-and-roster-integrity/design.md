## Context

当前队员管理后台在数据层、控件层和列表层同时存在缺口：

- 数据层只有 `birth_year`，没有完整生日；
- 控件层仍在使用原生日期输入包装；
- 列表层姓名与队伍信息重复，且桌面端列结构不够稳定；
- 编辑隐藏状态时存在明显的前端状态提交 bug。

这次设计需要同时保证三个目标：

1. 历史数据库安全升级；
2. 管理端获得完整生日能力；
3. 后台高频页面在视觉和交互上变得更稳定。

## Goals / Non-Goals

**Goals:**

- 支持后台完整生日录入与回显。
- 保持旧 `birthYear` 数据和旧客户端请求兼容。
- 采用 shadcn 官方推荐的 Calendar 组合来承载日期选择。
- 修复隐藏状态不可逆 bug。
- 优化桌面端花名册表格的信息结构和动作区。

**Non-Goals:**

- 不把完整生日暴露给公开接口。
- 不重做年龄桶、达级、成长分析等基于年份的现有算法。
- 不伪造旧数据的月日。

## Decisions

- **新增 `birth_date` 列，但不移除 `birth_year`**
  - `birth_date` 作为后台精确生日字段；
  - `birth_year` 继续保留，供现有年龄桶和兼容链路使用。

- **生日归一化放到 repository**
  - 创建 / 更新队员时统一解析 `birthDate` 与 `birthYear`；
  - `birthDate` 有值时，以它为准并派生 `birthYear`；
  - 只提供 `birthYear` 时仍兼容旧请求；
  - 历史没有 `birthDate` 的记录保持为空。

- **只在 admin payload 返回 `birthDate`**
  - public service 继续只输出已有公开字段；
  - 这样可以避免把更精细的个人信息泄露到公开页。

- **前端日期组件采用 `Popover + Calendar`**
  - 新建共享 `Calendar` primitive；
  - `DatePickerInput` 使用官方推荐结构；
  - 出生日期 placeholder 固定为“请选择”。

- **编辑状态可逆性在前端显式修复**
  - `hidden` 仍表示强制隐藏；
  - 但当管理员切回 `nickname` / `real_name` 并重新勾选公开时，提交载荷必须把 `isPublic: true` 一并送出。

- **桌面端表格显式收口列职责**
  - 姓名列只显示昵称 / 真名两层身份；
  - 队伍列单独显示；
  - 操作列删掉低价值下载动作。

## Risks / Trade-offs

- **生日字段双轨并存会增加状态复杂度**
  - 用 repository 单点归一化降低分叉风险。

- **新增日期依赖会触发前端锁文件变化**
  - 只补最小依赖，并通过现有 lint / test / build 验证。

- **修复隐藏状态时可能引入新的表单联动问题**
  - 增加页面级测试覆盖“隐藏 -> 公开”的真实编辑流。

- **完整生日仅后台可见可能让部分内部导出暂时还只看到年份**
  - 这是有意的边界控制；本轮目标是先修后台档案工作流，不扩大公开或导出范围。
